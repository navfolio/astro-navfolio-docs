import FriendLinkCard from '@navfolio/mdx-components/FriendLinkCard.astro';

`FriendLinkCard` 用于在 MDX 内容中展示友链。卡片会根据自身容器宽度而不是浏览器宽度决定列数：宽正文可以四列，带目录的文章正文通常会自然收为三列，再依次变为两列和一列。

## 基础用法

```astro
import FriendLinkCard from '@navfolio/mdx-components/FriendLinkCard.astro';
```

<FriendLinkCard
  items={[
    {
      name: 'Astro',
      url: 'https://astro.build/',
      bio: 'The web framework for content-driven websites.',
      avatar: 'https://astro.build/assets/press/astro-icon-light-gradient.svg',
    },
    {
      name: 'GitHub',
      url: 'https://github.com/',
      bio: 'Collaborative software development and source hosting.',
      avatar: 'https://github.githubassets.com/favicons/favicon.svg',
    },
    {
      name: 'Bun',
      url: 'https://bun.com/',
      bio: 'Fast JavaScript runtime and package manager.',
      avatar: 'https://bun.com/favicon.ico',
    },
  ]}
/>

`bio` 最多显示两行并自动省略，卡片维持稳定高度。没有 `backgroundImage` 时，头像会作为淡化背景；没有头像时，名称首字母会成为圆形占位头像。

## 数据来源与排序

```astro
<FriendLinkCard
  columns={4}
  randomBackground={true}
  shuffle={true}
  src="https://example.com/friends.json"
  items={[{ name: 'Fallback', url: 'https://example.com/' }]}
/>
```

默认 `source="auto"`：优先加载 `src`，远程 JSON 请求或解析失败时回退到 `items`。`source="src"` 强制只使用远程来源；`source="items"` 强制只使用内联数据。

`columns` 是容器足够宽时的最大列数，默认 `4`。实际列数会按卡片列表自身宽度调整。`randomBackground` 会忽略所有 `backgroundImage`，在服务端为每张卡片生成半透明随机背景色，因此首次绘制不会闪烁。`shuffle` 只打乱未置顶的项目。

## 友链字段

| 字段 | 说明 |
| --- | --- |
| `name` | 必填，站点名称。 |
| `url` | 必填，跳转地址。 |
| `bio` | 可选，最多显示两行。 |
| `avatar` | 可选，圆形头像；也会作为无背景图时的淡化背景。 |
| `backgroundImage` | 可选，指定卡片背景图。 |
| `rss` | 可选，友链朋友圈同步器使用的 RSS 或 Atom 地址。 |
| `sticky` | 可选，`true` 或正数；数值越大，越靠前。 |

友链朋友圈的 RSS 抓取与部署请见 [Friend Circle Sync](/blog/friend-circle-sync/)。
